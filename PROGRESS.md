## 技術スタック

- React + TypeScript + Vite
- 描画: Three.js（`OrthographicCamera` による2Dビューポート）
- 状態管理: Zustand（`src/scene/store.ts`）

## アーキテクチャ概要

- `src/scene/types.ts` — `Mesh`（vertices + faces）、`Transform`（位置・回転・スケール・ピボット）、`SceneObject` の型定義
- `src/scene/store.ts` — シーン全体の状態（オブジェクト一覧、選択状態、モード、Undo/Redo履歴）と全操作のアクション
- `src/scene/transformUtils.ts` — ローカル座標⇔ワールド座標の変換（ピボット基準）
- `src/scene/primitives.ts` / `objImport.ts` — 矩形・円の生成、OBJファイル読み込み
- `src/scene/extrude.ts` / `loopCut.ts` / `loopPath.ts` / `deleteElements.ts` — 編集操作のメッシュ変換ロジック（すべて「Mesh→Mesh」の純粋関数）
- `src/viewport/Viewport.tsx` — Three.jsでの描画とポインター/キーボード操作のハンドリング（このファイルが一番大きく、ビューポート内の全インタラクションを持つ）。`rebuildScene()`は毎フレーム全ジオメトリ/マテリアルを作り直す設計なので、`scene.clear()`の前に必ず`disposeSceneContents()`でdisposeすること（怠るとGPUメモリリークで時間経過とともに重くなり、最終的にブラウザが固まる）
- `src/panels/` — Toolbar・Outliner・Properties の各UIパネル
- `src/App.tsx` — グローバルなキーボードショートカット

## 実装済み機能

### モデリング基盤
- [x] プリミティブ生成: 矩形（X/Y分割数指定）、円（分割数指定）。編集モード中はBlenderの3Dカーソル相当の仕組みがないため、ゴースト枠がマウスに追従し、クリックで配置位置を確定する方式で、新規オブジェクトではなく現在編集中のメッシュに非連結の島として追加される（Esc/右クリックでキャンセル）
- [x] OBJファイル読み込み（平面メッシュ想定、軸自動検出、サイズ自動正規化）
- [x] 頂点・辺・面の選択編集（クリック選択、Shift複数選択。クリック自体はドラッグしても移動しない、選択のみ）
- [x] 頂点・辺・面の移動（Gキー。Blender風モーダル操作—クリック不要でマウス追従、クリック/Enterで確定、Esc/右クリックでキャンセル。確定時、トポロジー上隣接する頂点に重なったら自動でスナップマージ）
- [x] 押し出し（辺選択 / 頂点2点以上選択、Eキー。Blender風に、押し出した瞬間は元の位置に重なったままGキー相当のグラブモーダルに自動で入り、マウスでドラッグした分だけ押し出される。押し出し+移動は1回のUndoでまとめて取り消され、ドラッグ中のEsc/右クリックは押し出し自体を取り消す）
- [x] ループカット（トポロジーベースのクアッドストリップ走査、ホイールで複数本同時カット、Cmd/Ctrl+Rでツール切替）
- [x] 頂点・辺・面の削除（Delete/Backspaceキー。全面削除でオブジェクトごと自動削除）
- [x] 頂点マージ（Mキー→1=最初の頂点/2=最後の頂点/3=中間位置の選択順依存モーダル）
- [x] 面の穴埋め（Fキー。選択順そのままを多角形の頂点順にする。Ngonも作れる）
- [x] 複数選択（矩形選択。空白からドラッグで範囲選択、Shiftで追加。Cmd/Ctrl+Aで全選択）
- [x] ナイフツール（Kキー。辺/頂点をクリックして連結、Enter/ダブルクリックで確定、Esc/右クリックでパスをキャンセル。複数面を跨ぐ直線も自動で各面の交差点を検出して分割）
- [x] 編集モードでの要素の回転・拡大（編集モード専用ピボット。デフォルトはオブジェクトのピボットを継承、Pキーで選択要素の重心に再設置。R/SキーでBlender風モーダル操作—クリック不要でマウス追従、クリック/Enterで確定、Esc/右クリックでキャンセル。Rは差分回転、Sはピボットからの距離比率で均一スケール。常時表示はピボット位置の小さい点線リングのみ）
- [x] UVマッピング（`src/scene/uv.ts`。辺の共有（シームでない辺）でfaceをつないでisland単位で検出し、islandごとに自分のバウンディングボックスをアスペクト比を保ったまま正規化する自動展開（長い方の軸が0〜1、短い方はその比率分だけ小さくなる。X/Yを別々に正規化して正方形に押し込むことはしない）。2D平面メッシュなので手動アンラップは不要）。プロパティパネルの「UV」セクションにミニUVエディタ（`UvEditor.tsx`）があり、islandごとにドラッグで移動・右上角のハンドルでスケール調整可能（`SceneObject.uvIslandTransforms`、island順インデックスなのでトポロジーが変わると対応がずれる）。UVワイヤーフレームを指定解像度のPNG画像として書き出し可能
- [x] UVシーム（編集モード・辺選択でツールバーの「シーム」ボタン。トポロジーは繋がったままUVだけ分割できる。例: ワンスキンの人型で腕・脚をトポロジーは繋げたまま別UVislandに分割し、正方形UVへのパッキング効率を上げる。`SceneObject.seamEdges`、ビューポートでは赤線で表示。`findIslands`/`computeUVs`の第2引数にシームのSetを渡す方式（頂点インデックスに依存するため、トポロジーが変わるシーム後の操作で対応がずれる可能性あり）
- [x] UVのデフォルト初期配置（`defaultIslandTransforms`。手動でUVエディタを操作していないislandは、全部0,0に重なって出る代わりに、実寸の比率を保ったまま簡易グリッドに自動配置される。本格的なbin-packingではなく、グリッドの1セルに収めるだけの軽量な対策。手動で調整した分だけ上書きされる）
- [x] テクセル密度の一致トグル（UVエディタモーダル内のチェックボックス。ONの状態で1つの島をスケールすると、他の島も「scale ÷ 実寸(footprint)」の比率を保ったまま、各自の中心を軸に追従スケールする。体の部位ごとにテクスチャの精細さがバラつかないようにするための機能）。各islandの左下に鍵アイコンがあり、クリックで密度一致の対象から個別に除外できる（`UvIslandTransform.excludeFromDensityMatch`。除外した島は他の島のドラッグに追従せず、自身をドラッグしても他の島に影響しない。顔だけ意図的に高密度にしておくような用途）
- [x] UVエディタでisland単位の回転（`UvIslandTransform.rotation`、ラジアン、自身のbaseUVバウンディングボックス中心が軸。中心から固定ピクセル距離の丸ハンドルをドラッグして回転。斜めの腕・脚をX/Y軸に揃えることでUV空間の無駄を減らせる）。`normalizeIslandTransform`が`undefined`/`NaN`/`Infinity`をすべて弾いてフォールバック値に戻すので、フィールド追加前の古い保存データや計算ミスによる破損値が永続して伝播することはない（`setUvIslandTransform`の書き込み側でも不正な値はそもそも保存しない）
- [x] テクスチャ貼り付け（プロパティパネルの「マテリアル」セクションから画像ファイルを選択し`material.textureUrl`（Data URL）に設定。Three.js側は`computeUVs`の結果をジオメトリの`uv`属性に設定し、`MeshBasicMaterial.map`としてテクスチャを適用。読み込んだテクスチャはURLをキーにキャッシュして再利用し、`material.color`はテクスチャに乗算される（プロパティパネルにその旨を明記））
- [x] トレース用リファレンス画像（ツールバーの「🖼 下絵」。シーン全体で1枚、位置・スケール・不透明度を調整可能。グリッド・全オブジェクトより奥（z=-50）に描画。ツールバーの「メッシュ不透明度」スライダーで全オブジェクト共通の不透明度を下げ、下絵を透かして見ながらモデリングできる）
- [x] プロジェクトの保存/読み込み（独自拡張子`.fltd`、中身はJSON。`src/scene/project.ts`。テクスチャ・下絵もData URLとして埋め込まれるので1ファイルで自己完結。保存対象は`objects`・`referenceImage`・`meshOpacity`のみで、選択状態やUndo履歴は保存しない）
- [x] SDS（サブディビジョンサーフィス）（`src/scene/subdivision.ts`。Catmull-Clark分割を表示用のフィルメッシュにのみ適用し、編集用の低ポリcontrol cage・UVの実体は変更しない（`SceneObject.sdsLevels`、0〜3、プロパティパネルの「メッシュ」セクションで設定）。プロパティパネルの「SDSワイヤーフレームを表示」チェックボックスで、分割後の実ジオメトリのワイヤーフレームを確認可能（オンのときは元のcontrol cageも薄い点線で重ねて表示）。境界の頂点は corner-cutting 則で輪郭がなめらかに丸まり、3辺以上がシャープな頂点（角）は位置を固定する単純なフォールバックにしている）
- [x] SDSのエッジ/頂点クリース（重み0〜1）（`SceneObject.creaseEdges`/`creaseVertices`、辺キー・頂点インデックス→重みのレコード。編集モードで辺または頂点を選択し、ツールバーの波線アイコンのスライダーで選択中の全要素に一括設定（重み0で削除）。選択中の要素間で値が混在している場合はスライダー表示が「—」になるが操作は可能で、動かせば選択中全部に新しい値が一括適用される（Blender同様）。エッジクリースは辺の分割点の計算をスムース⇔シャープでブレンドし、頂点クリースは最終的な頂点位置を「ルール通りの位置」⇔「元の位置（不動）」でブレンドする（重み1で完全固定、毛先や角を尖らせたまま残せる））

## 今後のロードマップ（フェーズ分け）

開発順序の方針。理由: テクスチャがない状態でボーン変形を作ると、変形によるテクスチャの歪み・破綻を目で確認できず、後からテクスチャを貼った時点でボーンの入れ方が悪かったと分かる手戻りが起きやすい。なので「貼れる状態」を先に作ってからボーンに進む。

- **Phase 1（基本的な2Dモデリングツールとして完成）**: UVマッピング → テクスチャ貼り付け
- **Phase 2（DCCと呼べる状態に）**: ボーン・スキニング/デフォーム → 頂点グループ（スキニングウェイト割り当てのタイミングで実装） → コンストレイント（IK・揺れ物） → ボーンアニメーション
- **Phase 3（完了の定義に到達）**: アニメーションからのスプライトシート書き出し

現在地: Phase 1完了（UVマッピング・テクスチャ貼り付けまで実装済み）。次はPhase 2のボーン・スキニング。

## 既知の制約・設計判断（あえてそうしている）

- 面の三角形分割は先頭頂点からの扇形分割のみ。凹形（コンケーブ）ポリゴンは描画が崩れる可能性がある
- 頂点削除は「触れている面を全部消す」方式（Blenderの Dissolve Vertices のような面統合はしない）
- 辺/面/ループカット/押し出し/OBJ読込のいずれの操作後も、どの面にも使われなくなった孤立頂点は例外なく自動削除される（`pruneOrphanVertices`）
- 全頂点/全面を削除してメッシュが空になったオブジェクトは自動的に削除される（頂点を打って面を作る機能が無いため、空オブジェクトを残す意味がない）
- ループカットは四角面が連なっている場所でのみ機能する（三角形フチや非四角形は境界として扱われる）
- 面の穴埋め（F）・矩形選択は、選択順序の意味が違う（Fは選択順が多角形の頂点順になるので重要、矩形選択は頂点インデックス順になるので無関係）。組み合わせて使うと意図しない面ができるので、Fを使うときは1点ずつShiftクリックで順番に選ぶ必要がある
- ナイフツールのスナップ先は既存の辺上の点（頂点含む）のみで、面の内部に自由点は打てない。面は凸であることを前提に交差点を計算しているため、凹形（コンケーブ）ポリゴンを跨ぐカットは想定外の結果になる可能性がある

### 将来のボーン/スキニング実装に向けた方針（未実装・決定事項のみ記録）

- Ngon（5頂点以上の面）や凹形の面は、編集中は自由に作れる状態を維持する（Fキーでの穴埋めなど、ユーザーの意図を保つため禁止しない）
- ただし、ボーンへのバインド（スキニング）実行時に、メッシュ内にNgon/凹面が存在する場合はその旨を明示してユーザーに警告し、バインドを中断する（または三角形分割してから進めるかを選ばせる）。変形計算は三角形が常に一意に定まる形なので、変形用の内部表現は別途三角形分割したものを使う想定
- つまり「編集用メッシュ（Ngon/Quad混在可）」と「変形計算用メッシュ（内部で三角形分割済み）」を分離する方針。今すぐの実装は不要
