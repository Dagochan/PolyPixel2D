# PolyPixel2D 実装進捗

> 元の依頼仕様は [PolyPixel2D.md](./PolyPixel2D.md) を参照。
> このファイルは「今、何が実装済みで、何が未実装か」を追うための進捗まとめ。随時更新する。
> FlaFlo2D（前身アプリ、`git@github.com:Dagochan/2D_DCC_Tool` で開発継続中）のコードをベースに分岐し、不要機能を削った上で新規要素を追加している。

最終更新: 2026-06-28

## 技術スタック

- React + TypeScript + Vite
- 描画: Three.js（`OrthographicCamera` による2Dビューポート）
- 状態管理: Zustand（`src/scene/store.ts`）

## アーキテクチャ概要

- `src/scene/types.ts` — `Mesh`（vertices + faces）、`Transform`（位置・回転・スケール・Head）、`SceneObject`（`tail`・`parentId`・`connected`・`islandZOrders`・`islandNames`・`showIslandNames`を含む）の型定義
- `src/scene/store.ts` — シーン全体の状態（オブジェクト一覧、選択状態、モード、Undo/Redo履歴）と全操作のアクション
- `src/scene/transformUtils.ts` — ローカル座標⇔ワールド座標の変換（`getWorldTransform`/`getParentWorldTransform`で親子チェーンを再帰的に解決。ルートオブジェクトは従来通り`obj.transform`そのまま）
- `src/scene/meshUtils.ts` — 辺・三角形分割（`triangulatePolygon`、イヤークリッピングで凹形Ngon対応）などの基本ユーティリティに加え、`clampToMesh`（メッシュ輪郭への点のクランプ）、`localBoundsCenter`（アイランドのBBox中心）を追加
- `src/scene/dissolve.ts` — `dissolveVertices`/`dissolveEdges`（周囲の面を1つに結合してから削除）
- `src/scene/uv.ts` — `findIslands`でメッシュを連結成分（アイランド）に分割。`computeSplitUVIslands`でアイランドごとに個別のメッシュ+UVを生成し、`computeSplitUVs`はそれを結合するだけの薄いラッパー
- `src/scene/primitives.ts` / `objImport.ts` — 矩形・円の生成、OBJファイル読み込み
- `src/scene/extrude.ts` / `loopCut.ts` / `loopPath.ts` / `deleteElements.ts` — 編集操作のメッシュ変換ロジック
- `src/viewport/Viewport.tsx` — Three.jsでの描画とポインター/キーボード操作のハンドリング。アイランド単位の描画（`islandZOrders`に応じたZスタック）、Head/Tail（ピボットモード専用ハンドル＋オブジェクトモードの参照リング）、コネクターライン、アイランド名ラベル（canvas-textureのSprite）などをここで描画
- `src/panels/` — Toolbar・ToolPane・Outliner・Properties・UvEditor の各UIパネル
- `src/App.tsx` — グローバルなキーボードショートカット

## 実装済み機能

### モデリング基盤（FlaFlo2D から継承）
- [x] プリミティブ生成: 矩形（X/Y分割数指定）、円（分割数指定）。編集モード中はゴースト枠がマウスに追従し、クリックで配置位置を確定（現在編集中のメッシュに非連結の島として追加）
- [x] OBJファイル読み込み（平面メッシュ想定、軸自動検出、サイズ自動正規化）
- [x] 頂点・辺・面の選択編集（クリック選択、Shift複数選択、矩形選択）
- [x] 頂点・辺・面の移動（Gキー、Blender風モーダル操作）
- [x] 押し出し（Eキー）、ループカット（Cmd/Ctrl+R）、ナイフツール（Kキー）
- [x] 削除（Delete/Backspace、面を含めて消える）、ディゾルブ（Ctrl+X、周囲の面を1つに結合してから削除。`src/scene/dissolve.ts`）、頂点マージ（Mキー）、面の穴埋め（Fキー）
- [x] 面の三角形分割はイヤークリッピング方式（`triangulatePolygon`、凹形Ngonにも対応。メッシュ本体の塗りと面選択ハイライトの両方がこれを使用）
- [x] 編集モードでの要素の回転・拡大（R/Sキー、編集モード専用ピボット）
- [x] UVマッピング（アイランド単位の自動展開、ミニUVエディタでのドラッグ調整、テクセル密度一致機能）
- [x] テクスチャ貼り付け、トレース用リファレンス画像、プロジェクトの保存/読み込み（`.fltd`、JSON）
- [x] Undo/Redo、背景グリッド

### 階層構造・ペアレント（新規）
- [x] `parentId`によるオブジェクト間の親子付け。アウトライナーでドラッグ&ドロップして設定、空白部分へのドロップやリンク解除アイコンで解除
- [x] `connected`フラグ（デフォルトtrue）— trueの間は子のHeadが常に親のワールドTail位置にスナップ（典型的なボーンチェーン）。falseにすると独立した位置を保ちつつ回転・スケールの継承は維持し、親Tail→子Headの点線コネクターラインで関係を視覚化
- [x] 親子のワールド変換は`getWorldTransform`/`getParentWorldTransform`が再帰的に合成（循環参照は安全にルート扱いにフォールバック）
- [x] プロパティパネルの「階層」セクションで親の確認・解除・connected切替

### Head/Tailピボット（新規）
- [x] `Transform.head`（旧pivot）と`SceneObject.tail`の分離。Tailは子オブジェクトの接続先になるローカル座標点
- [x] 専用の「ピボットモード」を新設。オブジェクトモードではHead/Tailは中抜きの参照リングとして表示されるのみでドラッグ不可、ピボットモードに切り替えた時だけ塗りの円としてドラッグ編集可能
- [x] Head/Tailは常にメッシュ自身のシルエット内にクランプ（`clampToMesh`。BBoxではなく実際の輪郭・複数アイランドの境界も考慮した最近接点へスナップ）
- [x] プロパティパネルのHead X/Y・Tail X/Yフィールドからも数値編集可能

### アイランドZオーダー（新規）
- [x] 1オブジェクトの複数アイランド（連結していない複数の島）に個別の重なり順を設定可能。`islandZOrders`でランク管理、Viewportはアイランドごとに描画してZを微小オフセット
- [x] プロパティパネルの「アイランド」セクション（編集モード限定で表示、マテリアルセクションより上）— ▲▼で重なり順入れ替え、◎ボタンで該当アイランドを選択（編集モードに切替）、名前は直接編集可能（IME変換確定のEnterと誤反応しないよう対応、空にして確定すると自動でデフォルト名に戻る）
- [x] 「名前を表示」トグル（オブジェクト単位の一括ON/OFF）で、各アイランドのBBox中心より少し下にラベル表示。フォントはUIと同じシステムフォント、背景は角丸でビューポート背景に近いグレー
- [x] `selectLinked`（Ctrl/Cmd+L、Blenderの「Select Linked」相当）— 選択中の頂点/辺/面が属するアイランド全体に選択を拡張

### オブジェクトモードの操作性（新規）
- [x] World/Localギズモ基準切替（ツールバーの「ギズモ」ドロップダウン、オブジェクトモード限定）。ローカル（デフォルト）はオブジェクト自身の回転に追従、ワールドは常にシーンのXY軸基準。移動の軸矢印・BBoxアウトライン・スケールハンドルの位置が対象（回転リングは向きの概念がないので対象外）
- [x] 親に回転・スケールがあるオブジェクトを移動/回転させた時の「ぴょこっと」ジャンプ・ズレを修正（ワールド差分をそのままローカルtransformに代入していたバグ。`worldPositionToLocalOffset`で正しく変換するよう修正）
- [x] アウトライナーのドラッグ&ドロップを改善: 行の上端/下端＝同じ階層内で並び替え（zOrder、`reorder`アクション）、中央＝子としてペアレント、と判定。各行に専用のドラッグハンドル（⠿）を追加し、名前編集フォームとの誤操作を防止

### Emptyオブジェクト（新規）
- [x] メッシュを持たない、階層用のダミーオブジェクト（`SceneObject.kind: 'empty'`、`mesh`は空配列）。ツールバーの「+ Empty」で追加
- [x] ビューポートでは固定スクリーンサイズの十字＋リングのギズモとして表示（オブジェクトモードの移動/回転/スケールギズモは通常オブジェクトと共通利用、`getGizmoGeom`が空メッシュ時は固定サイズの仮想バウンディングボックスにフォールバック）
- [x] クリック選択は中心から一定ピクセル半径のヒット判定で対応（`pickObject`）。中心付近をクリックした場合は移動ギズモの軸矢印より自由ドラッグを優先するため、矢印のヒット判定に専用デッドゾーンを設けた
- [x] 編集モードには入れない（`setMode`でガード）。プロパティパネルもマテリアル/UV/メッシュ統計セクションを非表示にし、階層・トランスフォームのみ表示

### 選択・操作性の改善（新規）
- [x] 辺ディゾルブ（Ctrl+X）で、不要になった頂点（残りの辺が2本だけになった、連結していた辺がすべて消えた頂点）も自動削除。Blenderの「Dissolve Edges」の「Dissolve Vertices」相当（`src/scene/dissolve.ts`）
- [x] ループカット中、Shiftキーでカット位置をエッジのちょうど中央（t=0.5）にスナップ（1本カットの時のみ。マウスを動かさずキー押下時点で即反映）
- [x] 辺選択モードでのループ系選択を2種類用意: **Alt+クリック**＝エッジリング（面を挟んで反対側の辺へジャンプしていく平行な辺の並び、既存の`findFullLoop`を流用）、**Cmd/Ctrl+クリック**＝エッジループ（4本辺が集まる頂点を真っ直ぐ通り抜けて連続する辺の鎖、`findEdgeLoop`を新設）
- [x] 頂点モードで複数選択中の頂点をShiftなしでクリックした場合、辺・面モードと同様にその1点だけに選択を絞るよう統一（以前は誤って複数選択を維持していた）
- [x] 選択の反転（Cmd/Ctrl+I、`invertSelection`）。頂点/辺/面モードそれぞれで現在の要素タイプ内の選択⇄非選択を入れ替える
- [x] ビューポートのパンはミドルクリック（ホイールボタン）ドラッグのみに統一。Alt+ドラッグでのパンは廃止（Altはループ選択・頂点スライドのクランプ解除に使うため）

### アイデア（未確定・優先度低）: アイランドZオーダーの「挿入タグ」によるクロスオブジェクト割り込み

2026-06-27、ユーザーから提案された未実装のアイデア。アニメーションタイムラインの実装が一区切りついたあたりで検討する想定（先行実装はしない）。

**課題:** 例えば横向きキャラクターの「襟」は、前面・背面の2アイランドを持つ1オブジェクトとして自然にモデリングしたい。しかし「首」オブジェクトを襟の前面アイランドと背面アイランドの**間**に描画したい場合、現状の仕組み（オブジェクト単位のZオーダー→その中でアイランド単位のZオーダー、という2階層構造）では対応できず、従来のDCCツール同様に襟を前面・背面で2オブジェクトに分割するしかない。これはモデリング上のトポロジー編集（頂点結合・押し出しなど）を分断してしまうため避けたい。

**提案する仕組み:** 編集モードで、あるオブジェクトのアイランドZオーダーリスト上の「隣接する2つのアイランドの間」に挿入タグを置けるようにする。別のオブジェクトをそのタグに紐付けると、そのオブジェクト全体の描画位置が、タグの位置（=指定したアイランドの間）に割り込む。これにより「見た目の重なり順」と「メッシュ／オブジェクトの分割」を完全に分離できる。

**設計上のポイント（ユーザーとの会話で確認済み）:**
- 階層（親子・Head/Tail）とZタグは独立したレイヤーとして扱う。首が襟の親/子であることと、首がどこに描画されるかは別の関心事
- 現状の「オブジェクトZオーダー（1.0刻み）→アイランドランク（0.001刻み）」という2階層モデルのままでは実現できない可能性が高く、(a) オブジェクト+アイランドの組を単位とした単一のフラットな順序リストに作り変える、または (b) オブジェクトのZオーダーを「数値」だけでなく「{対象オブジェクトID, 対象アイランドindex, before/after}」のような相対指定でも表現できるようにする、のどちらかの方向性が必要になりそう。どちらを取るかは未決定で、着手時に検討する

**着手のタイミング:** ユーザー自身の提案により、アニメーションタイムラインの後で検討する。今は実装しない。

## 既知の制約・設計判断（あえてそうしている）

- アイランドの順序（`islandZOrders`/`islandNames`のインデックス）は`findIslands`が返す配列順（face探索順）に依存するため、トポロジーが変わる編集（頂点削除・マージなど）をすると対応がずれる可能性がある。`uvIslandTransforms`など既存のUV機能も同じ制約を持っており、それを引き継いだ形
- Head/Tailのクランプはメッシュの「現在の頂点位置」に対する最近接点を使うため、凹形メッシュや複数アイランドをまたぐ移動では一番近い境界に飛ぶことがある（意図した挙動）
- `connected`はルートオブジェクト（`parentId === null`）には意味を持たないが、フィールドとしてはデフォルトtrueのまま存在する。`getParentWorldTransform`は祖先を辿る際に`ancestor.parentId !== null`も同時に確認しており、ルート祖先のconnectedフラグが誤って子の位置計算に影響しないようにしている（過去に発生したバグの修正済み）
- ~~面の三角形分割は先頭頂点からの扇形分割のみ。凹形（コンケーブ）ポリゴンは描画が崩れる可能性がある~~ → イヤークリッピング方式への変更により解消済み（ディゾルブが凹形Ngonを作りやすいことを契機に修正）
- アイランド名ラベルはフレームごとに新しい`CanvasTexture`を生成するため、`disposeSceneContents`で明示的に破棄している（他のテクスチャはキャッシュ・再利用される前提のため、この特殊扱いが必要）
