> 元の依頼仕様は [FlaFlo2D_2D_DCC_Tool.md](./FlaFlo2D_2D_DCC_Tool.md) を参照。

最終更新: 2026-06-22

## 技術スタック

- React + TypeScript + Vite
- 描画: Three.js（`OrthographicCamera` による2Dビューポート）
- 状態管理: Zustand（`src/scene/store.ts`）

## アーキテクチャ概要

- `src/scene/types.ts` — `Mesh`（vertices + faces）、`Transform`（位置・回転・スケール・ピボット）、`SceneObject` の型定義
- `src/scene/store.ts` — シーン全体の状態（オブジェクト一覧、選択状態、モード、Undo/Redo履歴）と全操作のアクション
- `src/scene/transformUtils.ts` — ローカル座標⇔ワールド座標の変換（ピボット基準）
- `src/scene/primitives.ts` / `objImport.ts` — 矩形・円の生成、OBJファイル読み込み
- `src/scene/extrude.ts` / `loopCut.ts` / `loopPath.ts` / `deleteElements.ts` — 編集操作のメッシュ変換ロジック（すべて「Mesh→Mesh」の純粋関数）
- `src/viewport/Viewport.tsx` — Three.jsでの描画とポインター/キーボード操作のハンドリング（このファイルが一番大きく、ビューポート内の全インタラクションを持つ）
- `src/panels/` — Toolbar・Outliner・Properties の各UIパネル
- `src/App.tsx` — グローバルなキーボードショートカット

- [x] プリミティブ生成: 矩形（X/Y分割数指定）、円（分割数指定）
- [x] OBJファイル読み込み（平面メッシュ想定、軸自動検出、サイズ自動正規化）
- [x] 頂点・辺・面の選択編集（クリック選択、Shift複数選択）
- [x] 頂点・辺・面の移動（ドラッグ）
- [x] 押し出し（辺選択 / 頂点2点以上選択、Eキー）
- [x] ループカット（トポロジーベースのクアッドストリップ走査、ホイールで複数本同時カット、Cmd/Ctrl+Rでツール切替）

- [x] オブジェクトモード/編集モード切替（Tabキー）

- [x] ギズモは固定px表示（オブジェクトサイズに依存しない）、常に最前面表示（depthTest無効化）

### シーン管理
- [x] アウトライナー（一覧、表示/非表示、名前変更、Zオーダーのドラッグ並べ替え、削除）

## 既知の制約・設計判断（あえてそうしている）

- 面の三角形分割は先頭頂点からの扇形分割のみ。凹形（コンケーブ）ポリゴンは描画が崩れる可能性がある
- 頂点削除は「触れている面を全部消す」方式（Blenderの Dissolve Vertices のような面統合はしない）
- 全頂点/全面を削除してメッシュが空になったオブジェクトは自動的に削除される（頂点を打って面を作る機能が無いため、空オブジェクトを残す意味がない）
- ループカットは四角面が連なっている場所でのみ機能する（三角形フチや非四角形は境界として扱われる）
