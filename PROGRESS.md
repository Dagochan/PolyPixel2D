# PolyPixel2D 実装進捗

> 元の依頼仕様は [PolyPixel2D.md](./PolyPixel2D.md) を参照。
> このファイルは「今、何が実装済みで、何が未実装か」を追うための進捗まとめ。随時更新する。
> FlaFlo2D（前身アプリ、`git@github.com:Dagochan/2D_DCC_Tool` で開発継続中）のコードをベースに分岐し、不要機能を削った上で新規要素を追加している。

最終更新: 2026-07-11（スプライトシート書き出しを追加）

> 補足: 2026-06-29頃から2026-07-10の間、機能追加のたびにここを更新する運用が止まり、代わりにClaude Codeの自動メモリ（セッション横断の永続記憶）にのみ記録される期間があった。そのため間が空いているが、実装自体は継続していた。今後は両方を使い分ける: **PROGRESS.mdはコードと一緒にgit管理される「実装済み機能の説明」**、**メモリは「アイデア・フィードバック・進行中の経緯」**という役割分担。

## 技術スタック

- React + TypeScript + Vite
- 描画: Three.js（`OrthographicCamera` による2Dビューポート）
- 状態管理: Zustand（`src/scene/store.ts`）
- テストフレームワークは無し。検証は型チェック+scratchpadスクリプト+previewでの実操作で行う

## アーキテクチャ概要

- `src/scene/types.ts` — `Mesh`（vertices + faces）、`Transform`（位置・回転・スケール・Head）、`SceneObject`（`tail`・`parentId`・`connected`・`islandZOrders`・`islandNames`・`showIslandNames`・`modifiers`・`shapeKeys`などを含む）の型定義。`kind`で mesh/empty/path/lattice を区別
- `src/scene/store.ts` — シーン全体の状態（オブジェクト一覧、選択状態、モード、Undo/Redo履歴）と全操作のアクション。**Undo/Redoのスナップショットは`objects`と`clips`の両方を含む**
- `src/scene/transformUtils.ts` — ローカル座標⇔ワールド座標の変換（`getWorldTransform`/`getParentWorldTransform`で親子チェーンを再帰的に解決）
- `src/scene/meshUtils.ts` — 辺・三角形分割（`triangulatePolygon`、イヤークリッピングで凹形Ngon対応）、`clampToMesh`、`localBoundsCenter`、`findShortestVertexPath`（頂点間の最短経路選択、辺の実長で重み付けしたダイクストラ法）などの基本ユーティリティ
- `src/scene/composeDisplay.ts` — **`composeDisplayObjects`**: shape key → Fake Flag（頂点モード）→ Fake Physics（mesh）→ Path Deform (Rail) → FFD の順で変形を合成する共有パイプライン。Viewport.tsxとPixelPreview.tsxの両方がこれを呼ぶことで、変形の見た目が必ず一致する（過去にPixelPreview.tsxが独自コピーを持って一部の変形を反映し損ねていたバグ、および2026-07-10にViewport.tsx自身が同種の独自コピーを持っていたことが発覚し統合、の両方の教訓を踏まえた設計）
- `src/scene/shapeKeys.ts` — シェイプキー（モーフターゲット）。`displayVertices`でBasis+各キーのブレンド値を合成。Linear（直線ブレンド）/Arc（ピボット中心の弧に沿ったブレンド、大きな回転でのボリューム消失を回避）の2種類の補間モード
- `src/scene/fakeFlag.ts` — Fake Flag（sin波による旗・布のなびき）。回転モード（アンカー中心にオブジェクト全体が振り子のように揺れる）と頂点モード（頂点ごとにsin波で変位、布のような表現）の2種類
- `src/scene/fakePhysics.ts` / `fakePhysicsMesh.ts` — Fake Physics（揺れ物、バネによる遅延・オーバーシュート表現）。オブジェクトチェーン版（`parentId`で連結した実オブジェクトが親の動きに段階的に遅れて追従）とメッシュセクション版（1オブジェクト内の固定5セクションがカスケード）の2種類。どちらもクリップへのベイク（`simulateFakePhysicsChain`/`simulateFakePhysicsMeshSections`、ループ継ぎ目の`applyConvergence`収束処理込み）と、Viewport限定のライブドラッグプレビュー（`stepFakePhysicsMeshLive`、マウスでオブジェクトを動かした実時間dtでバネを毎フレーム積分）の両方に対応。メッシュセクション版はオブジェクト自身ではなく**祖先チェーンを含めて解決したワールド座標の動き**を揺れの発生源にする（`getWorldTransform`を時刻ごとに再評価）——頭にペアレントした髪メッシュのように、そのオブジェクト自身にキーフレームが無くても親の動きに反応する（2026-07-10修正、以前は自分自身のローカルキーフレームしか見ておらず無反応だった）。ループクリップは`simulateSpringLooped`でループ分の駆動信号を複数周ぶん助走させ、周期的な定常応答に収束させてから最後の1周を切り出す（2026-07-10追加、継ぎ目の不自然な強制ブレンドを解消）
- `src/scene/ffd.ts` — FFD（Free-Form Deformation）。専用の`kind: 'lattice'`オブジェクトをケージとして、バイリニア補間で参照先オブジェクトのメッシュを変形。ケージ自身のPath Deform (Rail)は畳み込んで反映するが、ケージのシェイプキー等（フレームごとの状態を要するもの）は`composeDisplayObjects`側から渡される`deformedVerticesById`マップ経由で反映（2026-07-10修正、後述）
- `src/scene/pathDeformRail.ts` — Path Deform (Rail)。Latticeケージ専用の2レール式パス変形、共通u→s写像で断面が常に垂直になるよう設計。旧・汎用メッシュ向けPath Deformは削除済み
- `src/scene/followPath.ts` — Follow Path。Blenderの同名コンストレイント相当、オブジェクトの`Transform`をPathに沿わせる（メッシュ自体は変形しない）。progressはキーフレーム可能、向きの解決はHead/Tailを流用＋Flipトグルで反転
- `src/scene/fakeBehind.ts` — FakeBehind。ステンシルバッファによる透明マスク（ロープが岩の後ろを通る、等の遮蔽表現）。Viewport・PixelPreview両対応
- `src/scene/gifDecode.ts` — アニメーションGIFのリファレンス画像対応（`gifuct-js`使用）。タイムラインplayheadに自動追従してコマ送り、Start offset調整、Flip X
- `src/scene/project.ts` — プロジェクトの保存/読み込み（JSON）。**保存拡張子は`.pptd`**（2026-07-10、前身アプリ由来の`.fltd`から改名。読み込み側は拡張子をチェックしないため旧`.fltd`ファイルもそのまま開ける）
- `src/scene/animation.ts` — アニメーションクリップ（`AnimationClip`）。`Transform`のキーフレーム、イージング（linear/easeIn/easeOut/easeInOut）、ループモード、Fake Physicsの各種ベイクトラックを保持
- `src/viewport/Viewport.tsx`（~4300行）— Three.jsでの描画と全ポインター/キーボード操作のハンドリング
- `src/viewport/PixelPreview.tsx` — ピクセルプレビューパネル。`composeDisplayObjects`を使い、メインビューポートと同じ変形結果を低解像度・ニアレストネイバーでレンダリング
- `src/panels/` — Toolbar・ToolPane・Outliner・Properties・Timeline・UvEditor の各UIパネル
- `src/App.tsx` — グローバルなキーボードショートカット

## 実装済み機能

### モデリング基盤（FlaFlo2D から継承）
- [x] プリミティブ生成: 矩形（X/Y分割数指定）、円（分割数指定）。編集モード中はゴースト枠がマウスに追従し、クリックで配置位置を確定（現在編集中のメッシュに非連結の島として追加）
- [x] OBJファイル読み込み（平面メッシュ想定、軸自動検出、サイズ自動正規化）
- [x] 頂点・辺・面の選択編集（クリック選択、Shift複数選択、矩形選択）。**頂点モードはCtrl/Cmd+クリックで最短経路選択**（直前に選択した頂点から辺の実長で重み付けした最短経路上の頂点をまとめて追加、2026-07-10）。**面モードも辺・頂点モードと同様にドラッグでの矩形範囲選択に対応**（2026-07-10、以前は面の内側から始まるドラッグが常に単一面クリックとして処理されていたバグを修正）
- [x] 頂点・辺・面の移動（Gキー、Blender風モーダル操作）
- [x] 押し出し（Eキー）、ループカット（Cmd/Ctrl+R、閉じた四角ストリップ/リングカットでジオメトリが重複するバグを2026-07-10に修正）、ナイフツール（Kキー）
- [x] 削除（Delete/Backspace、面を含めて消える）、ディゾルブ（Ctrl+X、周囲の面を1つに結合してから削除。`src/scene/dissolve.ts`）、頂点マージ（Mキー）、面の穴埋め（Fキー。**穴埋め後もVertexモードを維持するよう2026-07-10に修正**、以前は強制的にFaceモードへ切り替わっていた）
- [x] 面の三角形分割はイヤークリッピング方式（`triangulatePolygon`、凹形Ngonにも対応。メッシュ本体の塗りと面選択ハイライトの両方がこれを使用）
- [x] 編集モードでの要素の回転・拡大（R/Sキー、編集モード専用ピボット）
- [x] UVマッピング（アイランド単位の自動展開、ミニUVエディタでのドラッグ調整、テクセル密度一致機能）
- [x] テクスチャ貼り付け、トレース用リファレンス画像（アニメーションGIF対応含む、後述）、プロジェクトの保存/読み込み（`.pptd`、JSON）
- [x] Undo/Redo（`objects`と`clips`両方が対象）、背景グリッド
- [x] Object-mode Apply Scale（Scale X/Yをメッシュ頂点・Tail・UV・シェイプキー等にベイクし、スケールを1/1にリセット。外見は変化しない）

### 階層構造・ペアレント
- [x] `parentId`によるオブジェクト間の親子付け。アウトライナーでドラッグ&ドロップして設定、空白部分へのドロップやリンク解除アイコンで解除
- [x] `connected`フラグ（デフォルトtrue）— trueの間は子のHeadが常に親のワールドTail位置にスナップ（典型的なボーンチェーン）。falseにすると独立した位置を保ちつつ回転・スケールの継承は維持し、親Tail→子Headの点線コネクターラインで関係を視覚化（**非連結の子がオフセットを失う2箇所のバグを2026-07-06に修正**）
- [x] 親子のワールド変換は`getWorldTransform`/`getParentWorldTransform`が再帰的に合成（循環参照は安全にルート扱いにフォールバック）
- [x] プロパティパネルの「階層」セクションで親の確認・解除・connected切替

### Head/Tailピボット
- [x] `Transform.head`（旧pivot）と`SceneObject.tail`の分離。Tailは子オブジェクトの接続先になるローカル座標点
- [x] 専用の「ピボットモード」を新設。オブジェクトモードではHead/Tailは中抜きの参照リングとして表示されるのみでドラッグ不可、ピボットモードに切り替えた時だけ塗りの円としてドラッグ編集可能
- [x] Head/Tailは常にメッシュ自身のシルエット内にクランプ（`clampToMesh`）
- [x] プロパティパネルのHead X/Y・Tail X/Yフィールドからも数値編集可能

### アイランドZオーダー
- [x] 1オブジェクトの複数アイランドに個別の重なり順を設定可能。`islandZOrders`でランク管理、Viewportはアイランドごとに描画してZを微小オフセット
- [x] プロパティパネルの「アイランド」セクション — ▲▼で重なり順入れ替え、◎ボタンで該当アイランドを選択、名前は直接編集可能
- [x] 「名前を表示」トグル、アイランドごとの表示/非表示・ロックトグル
- [x] `selectLinked`（Ctrl/Cmd+L）— 選択中の要素が属するアイランド全体に選択を拡張

### オブジェクトモードの操作性
- [x] World/Localギズモ基準切替、G/R/Sキーボードショートカット対応
- [x] BBoxギズモに単軸スケールハンドル（辺の中点）を追加
- [x] アウトライナーのドラッグ&ドロップ（並び替え/ペアレント判定、専用ドラッグハンドル）
- [x] 角ハンドルドラッグでのスケール中、Shift押下中は元のScaleX:ScaleY比率を保ったまま拡縮（アスペクト比固定、2026-07-10）。ドラッグの主軸（より大きく動いた方）を基準に倍率を決定
- [x] **Volume Preserve モディファイア**（オブジェクトモード専用、2026-07-10）— Spine2D風のスクワッシュ&ストレッチ。片方の軸（Driving axis、X/Y選択可）を自分でキーフレーム/ドラッグし、もう片方を毎フレーム自動で逆比例計算（`otherScale = drivingMagnitude ^ -strength`）。Strengthダイヤルで無効(0)〜厳密な面積保持(1)まで調整可能、0.5だと柔らかい`1/√scale`相当。`composeDisplay.ts`/`Viewport.tsx`の変形チェーンに組み込み、ギズモ・Pixel Preview双方に反映

### Emptyオブジェクト
- [x] メッシュを持たない、階層用のダミーオブジェクト。固定スクリーンサイズの十字＋リングのギズモとして表示。編集モードには入れない

### Pathオブジェクト・パスアニメーション
- [x] `kind: 'path'`。制御点をクリックで配置、ドラッグで再配置、Enter/ダブルクリックで確定するベジェ風の裸のカーブオブジェクト（メッシュ非関与）。Closedトグル（Blenderの「Cyclic U」相当）
- [x] **Follow Path**（Blenderの同名コンストレイント相当）— オブジェクトの`Transform`をPathに沿わせる。progressはキーフレーム可能、向きはHead/Tail流用+Flipトグルで解決
- [x] **Path Deform (Rail)** — Latticeケージ専用の2レール式パス変形。共通u→s写像で断面が常に垂直になる設計。旧・汎用メッシュ向けPath Deformは設計を見直して削除済み

### FFD（Free-Form Deformation）
- [x] 専用の`kind: 'lattice'`オブジェクトをFFDケージとして参照し、バイリニア補間でメッシュを変形。ケージ自身がPath Deform (Rail)を持てば、参照側もそれに追従
- [x] ケージのトポロジー保護（列数/行数変更時はケージがリセット）、軸ロックスケール
- [x] **ケージのシェイプキーがFFD参照先に反映されないバグを2026-07-10に修正**。`composeDisplayObjects`を2段階構成（shape key/Fake Flag/Fake Physics適用済みの頂点を先に全オブジェクト分計算→FFDはそのマップからケージの頂点を読む）にリファクタリングし、あわせてViewport.tsxが持っていた`composeDisplayObjects`の独自コピーもこの共有関数を呼ぶよう統合

### シェイプキー（モーフターゲット）
- [x] オブジェクトごとに複数のシェイプキーを追加可能。「Sculpt」で頂点ツールによる孤立ポーズ編集、Basisとのブレンド重み（0〜1、多少のオーバーシュートも許容）
- [x] Linear（直線ブレンド）/ Arc（ドラッグ可能なピボット中心の弧に沿ったブレンド、大きな回転でのボリューム消失を回避）の2種類の補間モード
- [x] シェイプキーの値自体をタイムラインでキーフレーム可能
- [x] ブレンド重みスライダーは0/1付近（±0.04）でマグネットスナップ（2026-07-10、マウス操作で正確に0/1へ止めやすく）

### Fake Flag（旗・布のなびき）
- [x] 回転モード（アンカー中心にオブジェクト全体が振り子のように揺れる）と頂点モード（頂点ごとにsin波で変位、布のような表現）の2種類
- [x] Blender風のモディファイアスタック概念を新設（`SceneObject.modifiers`）

### Fake Physics（揺れ物）
- [x] オブジェクトチェーン版 — `parentId`で連結した実オブジェクトが、親の動きに段階的に遅れて追従（バネの減衰・オーバーシュート）。Bake（チェーンのROOTのみボタン表示）、ループクリップの継ぎ目収束ブレンド
- [x] メッシュセクション版 — 1オブジェクト内の固定5セクション（頂点グループ）がカスケード。Viewport限定のライブドラッグプレビュー（マウスでオブジェクトを動かした実時間dtでバネを毎フレーム積分）＋クリップへのベイクの両対応

### FakeBehind（遮蔽表現）
- [x] ステンシルバッファによる透明マスク（ロープが岩の後ろを通る、等）。パスアニメーションとは別機能。メインビューポート・Pixel Preview両対応

### 面カラー（Per-face color）
- [x] 面単位で個別の色を割り当て（Assign/Clear）、色ごとの選択リスト、外周のみの選択枠

### アニメーションタイムライン
- [x] 画面下部のタイムラインペイン（ドープシート風レイアウト）。1プロジェクトで複数のアニメーションクリップを保持・切り替え可能
- [x] `Transform`のキーフレーム、秒単位の時間軸、イージング（linear/easeIn/easeOut/easeInOut）
- [x] 再生トランスポート・プレイヘッドハンドル、前後キーフレームへのジャンプ、キーフレームの複製
- [x] クリップのフレームレートに量子化した再生（低fpsクリップが実際にコマ送りに見える）
- [x] 削除されたオブジェクトのキーフレームを全クリップから自動クリア

### グリッド・スナップ
- [x] サブグリッド表示（`gridSubdivisions`で分割数調整）
- [x] グリッドスナップ（ツールバーのチェックボックス、Ctrlで一時反転）。オブジェクトの移動、編集モードのG移動に対応

### アウトライナー・UI改善
- [x] 階層折りたたみ、特殊オブジェクト種別（Empty/Path/Lattice）の色分け・記号表示
- [x] プロパティパネルの初期高さ調整

### ピクセルプレビュー・下絵ベイク・パレット量子化
- [x] ピクセルプレビューパネル。低解像度・ニアレストネイバーでシーンの塗りのみをレンダリングし、`image-rendering: pixelated`で拡大表示。固定Pixel Frame（安定したフレーミング）or 自動フレーミング、解像度調整可能

## 既知の制約・設計判断（あえてそうしている）

- アイランドの順序（`islandZOrders`/`islandNames`のインデックス）は`findIslands`が返す配列順（face探索順）に依存するため、トポロジーが変わる編集をすると対応がずれる可能性がある
- Head/Tailのクランプはメッシュの「現在の頂点位置」に対する最近接点を使うため、凹形メッシュや複数アイランドをまたぐ移動では一番近い境界に飛ぶことがある（意図した挙動）
- `connected`はルートオブジェクト（`parentId === null`）には意味を持たないが、フィールドとしてはデフォルトtrueのまま存在する
- アイランド名ラベルはフレームごとに新しい`CanvasTexture`を生成するため、`disposeSceneContents`で明示的に破棄している
- Fake Physicsのライブドラッグプレビューは、実際の描画フレームごとの生の動き（マウスの動き＝高周波の入力）を追いかけるため、キーフレーム補間された滑らかな動きでの実際の再生よりも大きく揺れる（バネの計算式自体はdt非依存だが、入力信号の質が異なるため）。加えてループクリップの後半は`convergeStart`による収束処理で意図的に大人しくなる。数値を追い込むなら手動ドラッグではなく実際のキーフレーム再生で確認するのが望ましい
