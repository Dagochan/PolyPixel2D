# 開発依頼仕様書: プロジェクト名 : PolyPixel2D

> これは Claude Code への開発依頼仕様書です。未定・指定なしの項目は、Claude の判断で適切に決めて構いません。不明点があれば実装前に質問してください。

作成日: 2026-06-27

## 1. プロジェクト概要

- 一言で言うと: 2Dポリゴンで作ったオブジェクトに、Head（回転軸及び、親オブジェクトとの接続位置）とTail（子オブジェクトとの接続位置）のピボット仕様を持たせて連結できる。それらを組み合わせて間接付きのキャラモデルなどを構築していき、アニメーションさせる。いわゆる、よくあるボーンアニメーションのクサビ型ボーンボディをそのまま個別オブジェクトに置き換えたような仕様。最終的にはピクセライズしてドット絵の見た目でプレビュー、スプライトシートの書き出しをしたい。
- 想定ユーザー: アニメーション用の手打ちドット絵の下敷きを作りたい人、従来のピクセルベースのドット絵制作ではパーツの回転でシェイプがガタガタになるのに嫌気がさしている人向けのツール。
- 参考イメージ: 最終目標がドット絵作成という点ではAsepriteと似たイメージだけど、内部コンセプトが全く異なる。

## 2. 機能要件

### 必須機能（MVP）
- 各オブジェクトのペアレント機能。
- 先述のHead&Tailのピボット概念。HeadとTailのピボットは任意の位置に設定・変更が可能。
- XY平面特化型の2Dポリゴンモデリング
- 各オブジェクトのZオーダー指定。
- オブジェクトの階層表示が可能なアウトライナーパネル。
- アニメーション制作のためのタイムライン。
- undo/redo
- BBoxでのトランスフォーム編集機能。
- Zオーダー最下にトレースなどに使える下絵の読み込み表示機能。
- Blenderライクな各モード概念とポリゴン編集機能。
- オブジェクト単位で複数アイランド・トポロジーのメッシュを持つことが可能で、その際に各アイランドにZ位置で前後の重なり順を設定可能。

### あれば嬉しい機能
- テクスチャリング機能。
- IKなどのコンストレイント。
- リギング用のカスタムシェイプの運用。
- シェイプキーなどのモーフターゲット機能。

### やらないこと（スコープ外）
- 基本的にはパペットアニメーションのような仕組みにしたいので、ポリゴンを用いるがウエイトスキニングのような事はやらない。

## 5. 制約・運用方針

- 開発の位置づけ: 長期的に保守・拡張していく本格的なアプリ
- テスト・品質への要望: 特になし
- その他の制約: 当アプリにかなり似た構造を持つ「FlaFlo2D」というアプリを現在進行形で開発しているので、そのプロジェクトファイルをベースに当アプリの仕様に変更していきたい。SDSや、クリース、UVシーム分割などは今回不要と思われるので削る。一度に全て進めなくていいので、まずは削る部分から初めて、順序立てて新しい要素を足していく。
